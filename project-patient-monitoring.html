Inspiration

Healthcare workers are often stretched thin, monitoring numerous patients simultaneously. When a patient's condition deteriorates, every second counts. We recognized that combining affordable hardware with intelligent alert systems could save lives while reducing healthcare infrastructure costs. Our goal was to create a system that provides both immediate bedside feedback for nurses and centralized oversight for care teams.

What It Does

Healthy Connection monitors patient vitals in real time using ECG, temperature, and humidity sensors. Color-coded LEDs at each bedside provide instant visual feedback: green (vitals safe), yellow (attention needed), red (critical alert), and blue (system connected). Multiple patient monitoring units connect wirelessly via Bluetooth to a central hub built on the nRF7002 DK board, which transmits all data to our cloud-based web dashboard.

Healthcare providers can monitor all patients simultaneously, view real-time vital signs, track trends, and receive instant alerts when thresholds are exceeded. Patient IDs are entered and linked to individual profiles through the web interface. The architecture supports unlimited patient units connecting to one hub, enabling scalable deployment at significantly lower cost than traditional systems.

Challenges We Overcame

Voltage Compatibility Crisis: The nRF7002 DK board operates at 1.8 V by default, but our sensors required 3.3 V. We had to carefully reconfigure the board's voltage output to ensure proper sensor operation without damaging sensitive components‚Äîa critical early obstacle that required understanding hardware specifications and power management.

Wi‚ÄëFi Connection Stability: Establishing reliable Wi‚ÄëFi connection from the central hub to cloud services required extensive debugging. We struggled with intermittent connectivity, configuration challenges, and ensuring data transmission wouldn't drop during critical moments. Through persistence and systematic troubleshooting, we achieved stable communication.

Bluetooth Device-to-Device Communication: Setting up Bluetooth connections between patient monitoring units and the central hub involved troubleshooting pairing protocols, maintaining reliable connections, and handling multiple simultaneous devices. We ensured patient units could seamlessly connect without manual intervention for each device.

ECG Signal Processing Complexity: Obtaining clean, accurate ECG readings proved challenging due to electrical noise and signal quality issues. We implemented signal filtering algorithms to clean the data and established proper threshold detection for heart rate monitoring, balancing sensitivity with noise rejection.

Sensor Integration and Protocols: Developing code for temperature and humidity sensors required understanding sensor-specific communication protocols, timing requirements, and data formatting. Integrating multiple sensor streams into unified patient profiles demanded careful embedded and backend software architecture.

Learning Curve with New Hardware: Neither team member had prior experience with the nRF7002 DK platform. We learned the Nordic SDK, wireless protocols, and board configuration entirely during the 48‚Äëhour hackathon through documentation, experimentation, and rapid iteration.
